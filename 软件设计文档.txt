# 软件设计文档：CLI 文件自动分类与归档工具 (FileArchiver)

## 1. 概述 (Overview)
**项目名称**：FileArchiver (可自定义)
**开发语言**：Go 1.20+
**运行环境**：跨平台 (Windows / macOS / Linux)
**系统定位**：一个轻量级的命令行界面（CLI）工具，旨在帮助用户快速将散落的工作文件（如桌面文件）按预设分类、标准化命名规则并自动生成版本号，以“纯复制”的模式安全地归档到指定的集中存储目录中。

## 2. 系统架构与工作流 (System Architecture & Workflow)
系统采用单向数据流设计，核心流程分为四个阶段：
1. **参数解析层**：接收用户通过 CLI 输入的源文件路径、目标分类以及可选的命名模板。
2. **校验与映射层**：验证文件是否存在，检查分类名称是否合法，并将中文分类映射为标准英文缩写（如“教学” -> “EDU”）。
3. **版本计算层**：扫描目标分类的归档目录，基于正则表达式进行模式匹配，计算当前文件的最新版本号。
4. **持久化执行层**：在内存中渲染新的文件名，并通过流式 I/O 将源文件安全地复制到目标目录。

## 3. 功能需求设计 (Functional Requirements)

### 3.1 核心功能
* **标准化重命名**：根据模板动态生成包含分类缩写、日期、原文件名和版本号的新名称。
* **自动化分类**：根据用户输入的分类，自动在归档根目录下创建或定位对应的子文件夹。
* **智能版本控制 (Auto-Versioning)**：
    * 当归档目录中无同名（指基础文件名相同）文件时，版本号从 `v1` 开始。
    * 当归档目录中已存在历史版本时，自动识别最大版本号并递增（如 `v1`, `v2` -> `v3`）。
* **非破坏性归档**：采用 Copy 策略，仅将文件复制到归档池，原路径文件保持不变，保障用户当前工作流不被中断。

### 3.2 配置文件与环境变量
* **归档根目录**：目前在代码中以常量 `archiveBaseDir` 定义（建议未来迭代为读取环境变量或本地配置文件）。
* **分类字典映射**：内存级 Map 结构，支持横向扩展。

## 4. 核心逻辑与模块设计 (Core Logic Design)

### 4.1 模板引擎模块
采用轻量级的字符串替换机制处理命名规则，内置以下合法占位符：
* `{category_abbr}`：分类的英文缩写。
* `{yyyymmdd}`：执行任务当天的日期（如 20260409）。
* `{filename}`：剥离扩展名后的原始文件名。
* `{version}`：由系统自动计算的纯数字版本号。
* `{ext}`：原始文件扩展名（包含 `.`）。

### 4.2 版本探测模块 (`getNextVersion`)
* **匹配规则**：不依赖文件创建时间，而是严格依赖文件名中的版本标识。
* **正则匹配**：使用动态构建的正则表达式 `-{filename}-v(\d+){ext}$` 扫描目标目录。使用 `regexp.QuoteMeta` 对原文件名进行转义，防止特殊字符（如括号、加号）导致正则崩溃。

### 4.3 安全复制模块 (`copyFile`)
* 使用 `os.Open` 和 `os.Create` 获取文件句柄。
* 使用 `io.Copy` 进行底层数据流传输，避免将大文件一次性载入内存，保证对超大文件（如视频）的支持。
* 使用 `File.Sync()` 强制将系统内核缓冲区的数据刷入磁盘，确保断电等极端情况下的数据完整性。

## 5. 接口与参数说明 (CLI Interfaces)

程序通过标准 `flag` 包接收参数：

| 参数缩写 | 全称映射 | 必填 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `-f` | filepath | **是** | 无 | 需要被归档的源文件绝对或相对路径。 |
| `-c` | category | **是** | 无 | 文件的业务分类（如：教学、财务、开发）。 |
| `-t` | template | 否 | `{category_abbr}-{yyyymmdd}-{filename}-v{version}{ext}` | 自定义命名模板，一般无需修改。 |

**调用示例**：
```bash
./archiver -f "/Users/Admin/Desktop/Project_Plan.pdf" -c 开发
```

## 6. 异常与错误处理 (Error Handling)

系统针对以下边界情况进行了防御性编程：
* **输入缺失**：未提供 `-f` 或 `-c` 时，阻断执行并打印标准用法提示。
* **未知分类**：输入的分类不在 `categoryMap` 字典中时，提示“未知的分类”并安全退出。
* **源文件异常**：源文件不存在或权限不足导致无法读取时，抛出 I/O 错误。
* **目录创建失败**：归档根目录无写权限时，抛出权限错误，中止归档。

## 7. 未来迭代路线 (Future Roadmap)
为了增强工具的灵活性，以下功能可作为下一阶段（v2.0）的开发目标：
* **外部配置文件集成**：引入 `spf13/viper`，支持通过 `config.yaml` 外部定义 `archiveBaseDir` 和 `categoryMap`，实现代码与配置彻底解耦。
* **批量处理模式**：支持传入一个包含多个文件的目录作为参数，程序遍历并批量进行归档。
* **系统右键菜单集成**：编写对应的 Registry 脚本（Windows）或 Automator 脚本（macOS），将该 CLI 封装为图形化操作。
